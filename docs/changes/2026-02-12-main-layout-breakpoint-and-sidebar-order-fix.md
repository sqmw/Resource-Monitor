# 2026-02-12 Main Layout Breakpoint And Sidebar Order Fix

## Summary
- 修复主窗口顶部“设置栏抢占头部”导致的视觉别扭问题。

## Root Cause
- 响应式断点触发过早，窗口稍窄就切到单列。
- 单列时设置栏被放到最上方（`order: -1`），造成页面上半部被设置区占满。

## Changes
- 文件：`src/routes/+page.svelte`
- 将单列断点从 `1140px` 下调到 `980px`，常规窗口保持左右双栏。
- 单列时设置栏顺序改为在主内容之后（`order: 2`）。
- 调整侧栏列宽为 `272px`，并收敛单列时最大高度。

## Result
- 常规窗口下顶部结构恢复自然，不再出现“上方一整块设置区”。
- 小屏仍可用，且阅读顺序更符合“先看监控，再调设置”。
